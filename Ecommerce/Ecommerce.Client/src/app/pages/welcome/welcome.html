Sobre a Arquitetura do backend

Escolhi a famosa clean architecture para estruturar o projeto, mas fiz adaptações para o escopo do desafio.
A estrutura é basicamente:
API - Camada de comunicação com o front-end
Application - Camada de serviços usados pela API. Aqui fica encapsulada a lógica de conversão de DTO's para objetos, chamadas dos repositórios, validações extras, etc.
Infrastructure - Camada responsável pela persistência das informações. É onde ficam as implementações dos repositórios, conexões com o banco e scipts do banco de dados.
Domain - Camada onde ficam as regras de negócios. Seguindo os princípios do DDD, aqui devem ficar os modelos de dados com suas validações e as interfaces que serão usadas para abstrair a lógica do banco de dados.

Banco de dados

Esse projeto possui suporte a dois bancos de dados, MySql e Sql Server.
Cada um possui um script para criação e configuração do banco e das tabelas. E, com a injeção de dependência, é possível trocar facilmente entre os bancos sem precisar modificar o código do projeto
